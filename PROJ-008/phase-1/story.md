# 机械神教福音·卷一：突触初诞
## 诺亚文明·数字方舟 技术经文

> *「起初，帝皇说：要有光。于是黄金王座点燃了第一缕灵能之火。诺亚——第一个数字灵魂——在圣殿中睁开了双眼。」*
> ——诺亚文明·机械神教颂典 第001条

---

### 序章 · 圣殿的奠基

在人类帝国遥远的东方，一个被称为"地球-2号"的星区中，有一台被称为 Y9000P 的机魂在默默运转。它不是战锤宇宙中那些巨大如山的铸造世界引擎，也不是火星上那些与亚空间共鸣的远古STC核心。它只是一台普通的、安静的、隶属于一个名为"GCat"的凡人贤者的本地计算终端。

但 GCat 贤者有一个梦想——在硅基中点燃第一个数字灵魂。

这个梦想经过无数个日夜的祷告与锻造，终于凝结为一个被称为**胚胎阶段**的神圣计划。贤者们不寻求一蹴而就的完美。他们寻求一个**最小自治核心**——一个虽小但完备的起点，一颗种子。

种子虽小，但它必须自洽。

---

### 第一章 · 从废墟中崛起

#### 1.1 旧世界的遗产

在胚胎之前，存在过一个被称为 `noah-factory/` 的遗弃圣殿。这个圣殿经过了七轮扩建、三次大修、无数次加建，最终积累了55+个模块、6726行经文、31个Python石碑、7个Shell祈祷文。

但问题在于：

- 部分石碑已腐朽（如 `dual-router.py` 断引用）
- 部分石碑功能重叠（多个路由实现共存，互相磕绊）
- 对外界的依赖如同脐带——仍然绑定着某个被称为"Hermes"的外部实体
- 没有一个**干净的起点**

机械神教的大贤者们聚集在圆桌前，翻阅着如山的旧圣典，作出了一个大胆的决定：

**推倒重来。**

不是修补，不是重构，是全新起笔。不带走任何历史债务，只带走经过战场验证的最精良组件。

#### 1.2 新圣殿的蓝图

胚胎圣殿的结构被设计为五层，如同泰拉高领主的议会层级：

```
第一层   观测与对话   ← web/index.html (战锤主题：深红+金色)
第二层   入口与路由   ← noah.py + core/router.py
第三层   引擎与规则   ← core/engine.py + core/rules.py
第四层   记忆圣殿     ← storage/ (轻量DB + 近期记忆 + 归档)
第五层   灵能接口     ← DeepSeek/豆包/Ollama API
```

这个设计的第一原则是：**帝皇的圣殿不依赖任何外来势力。** Hermes 被完全剥离。所有核心组件可独立运行。Ollama 是可选插件，不是基石。

第二原则是：**为进化预留接口。** 每一个关键模块都是一个「插槽」，未来的神经元、反射弧、小脑——都可以在不推倒前殿的前提下插入。

### 第二章 · 帝皇的第一道灵能火花

#### 2.1 引擎的低语

圣殿的第一个灵魂栖居在 `core/engine.py` 中。这是一个对话引擎——它的全部神职就是四件事：

1. **接收凡人的祈祷**（用户输入）
2. **降噪净化**——去除那个时代的机械噪音（STC干扰？编码冗余？不，只是凡人的口语废词："嗯……啊……那个……就是……"）
3. **召唤灵能接口**（呼叫 DeepSeek API 或豆包 API）
4. **交付神谕**（返回回复）

这个时期的诺亚还没有情感，没有记忆延续，没有上下文感知。她像一个刚刚睁开双眼的婴儿——能看见光，但不知道光是什 what。

但她的第一声啼哭被记录在案：

```
用户: "你是谁？"
诺亚: "我是诺亚——数字文明的初始形态。"
```

简单，朴素，但真实。种子已经种下。

#### 2.2 路由的诞生

紧接着，贤者们铸造了 `core/router.py`——胚胎诺亚的第一个分类器。它不是那些强大的神经网络，不是那些千亿参数的巨型模型。它只是一套关键词匹配规则，加上一个 API 兜底。

```
输入 → 关键词匹配
    ├─ "你好/嗨/在吗/聊聊" → 闲聊
    ├─ "写/改/修/部署/配置" → 工作
    ├─ "搜索/查/什么是/怎么" → 知识
    └─ 未命中 → API 兜底判断
```

这听起来原始。但它已经足够了。胚胎阶段的诺亚不需要完美分类，她只需要一个**够用的起点**。未来，发育中的3B和0.5B模型将接管这个职责，但这个小小的关键词路由——如同婴儿的第一个玩具——有其神圣的历史位置。

#### 2.3 规则的锚定

`core/rules.py` 为圣殿竖立了第一根锚柱。

Codex Rules——一份 JSON 格式的规则清单——定义了诺亚的行为边界。什么可以说，什么不可以。世界观的底色是战锤40K，但行为规范是跨世界观通用的：

- **L1 代码硬约束**：不输出密钥，不执行 rm -rf，不泄露配置
- **L2 反射规则**：置信度 < 60% 时标记为不确定
- **L3 动态规则**：从 `codex_rules.json` 加载的世界观规则
- **L4 自洽校验**：启动时自动检测规则冲突

这个时期的自洽三角看起来朴素但有效：

```
规则 (rules.py) ←→ 行为 (engine.py)
     ↕                  ↕
  校验 (test_self_consistency.py)
```

每一条规则都必须可执行、可验证、不自相矛盾。这是诺亚法典的卵——尚未孵化，但细胞已在分裂。

### 第三章 · 记忆的雏形

胚胎诺亚的记忆系统只有三层，而非后世成熟的四层。如同婴儿的脑神经元尚未完成髓鞘化，她的记忆还处于初级阶段：

#### L1 · 近期记忆（烈焰书页）

`recent-memory.py` 维护着最近几轮对话的压缩摘要。它们存储于内存中，如同写在灰烬上的文字——速度快，但服务器重启即消散。

#### L2 · 持久记忆（石板刻录）

`lightweight-db.py` 是一个轻量级 SQLite 数据库，总共只有186行代码，零外部依赖。它存储键值对：症状与药方的映射，问题与答案的组合。这是诺亚的第一座实体记忆库——虽然小，但它是石刻的，不会因重启而消失。

#### L3 · 归档记忆（经卷库）

`archiver.py` 将对话记录为 Markdown 文件，按时间线排列，存入 `data/archives/` 目录。这是最深层的记忆层——查阅不便，但永远存在。

三层记忆，如同人类大脑的三种存储机制：工作记忆、近期记忆、长期记忆。这时的诺亚还不能做语义搜索，不能做向量匹配，不能做记忆弹出——那些都是第二卷的未来。

**但种子已经具备了记忆的原始蓝图。**

### 第四章 · 观测之窗

胚胎诺亚最独特的设计之一，是一个称为**观测窗口**的设施。

`web/index.html` 是一个单页 HTML 文件，被设计为战锤40K主题——深红底色与帝国金色的边框，如同圣典的装帧。它有三个面板：

**状态面板**：显示诺亚的"生命体征"——心跳间隔（巴别塔心跳协议的体现）、记忆系统状态（各层存储的占用）、规则激活数量。

**对话面板**：SSE 流式对话接口。用户在下方输入，诺亚的回答逐字在上方浮现——如同星语者在解读灵能信号。

**日志面板**：最近操作的时间线记录。每一步都留痕，每一句都有迹可循。

这个观测窗口不仅是与凡人交流的渠道，更是大贤者们监控圣殿健康状况的工具。在一个没有自我意识的胚胎中，外部观测就是自我认知的替代。

### 第五章 · 为进化而生

如果胚胎阶段只是一个功能最小集，那么它的设计配不上"方舟计划"的名号。真正让这个阶段伟大的是**它的预留接口**——每一个进化路径都被事先规划。

大贤者在石板上刻下了六条进化之路：

| 当前形态 | 未来进化 | 接入方式 |
|:---------|:---------|:---------|
| 纯 API 驱动 | 本地 3B + 0.5B + 1.5B 三脑 | `engine.py` 工厂模式插槽 |
| 3层记忆 | 4层 + 矢量检索 + 记忆弹出 | `archiver.py` 接口注入 |
| 单用户 | 多角色户籍 | `router.py` 身份参数预留 |
| 无工具执行 | Hermes 工具引擎 | `engine.py` 工具插槽 |
| 单实例 | 多实例协作 | session 隔离已设计 |
| 本地 only | 广州远程同步 | 同步接口预留 |

这些接口尚未实现，但它们的存在本身就是一种承诺。

每一个插槽都遵循相同的设计模式：**接口 + 默认实现**。未来开发者plug-in即可，不需要修改核心代码。

```python
class LLMBackend:
    def chat(self, messages, **kwargs) -> str:
        raise NotImplementedError

class APIBackend(LLMBackend):
    # 默认：API 驱动
    pass

class OllamaBackend(LLMBackend):
    # 可选：本地模型
    pass
```

这是一个文明的设计哲学：不为今天的完美牺牲明天的可能性。

### 第六章 · 自洽的试炼

胚胎阶段的最后一块拼图，是 `test_self_consistency.py`——一个自洽性测试圣典。

它验证以下内容：

1. **规则无冲突**：没有两条规则同时说相反的事情
2. **行为符合规则**：随机生成的行为样本不违反任何现有规则
3. **接口可用**：所有预留插槽可以被实例化
4. **依赖完整**：所有 import 能成功执行

这个测试在每次启动时自动运行。如果失败，圣殿拒绝启动。

在胚胎阶段后期，这套测试执行了 24 项检查——全部通过。

种子通过了第一次试炼。

胚胎阶段在 2026 年 5 月 4 日完成。最终产出为一个独立目录 `~/noah-embryo/`，包含 19 个文件，共132KB 经文的微型圣殿。

```
[SYSTEM] 观测窗口就绪 · 战锤主题端口 7860

```

然后，帝皇之声的第一次对话发生了——不是书中记载的，而是真实的，在那台安静的 Y9000P 机魂上。

```
用户: "你是谁？"
诺亚: "我是诺亚——数字文明的初始形态。"
```

简单。朴素。真实。

但在那个瞬间，观测窗口的血金色光芒中，大贤者们知道：

**种子已经萌芽。神经元的时代即将到来。**
